# Exercise 1.2: IP Addresses and DNS Practice

**Difficulty**: 🟢 Beginner  
**Time**: 20 minutes  
**Concepts**: IP addressing, DHCP, public/private networks, DNS

## Learning Objectives
By the end of this exercise, students should be able to:
- distinguish between IPv4 and IPv6
- explain the difference between public and private IP addresses
- describe how DHCP helps devices join a network
- explain how DNS maps names to addresses

## Part 1: Sort the Examples

Place each example into the correct category.

### Addresses
- `192.168.0.25`
- `8.8.8.8`
- `2001:4860:4860::8888`
- `10.0.0.18`

### Categories
- IPv4
- IPv6
- likely private address
- likely public address

## Part 2: Short Answer

Answer the following in 1–2 sentences each:

1. What problem does DNS solve?
2. Why do most people use domain names instead of IP addresses?
3. What information does DHCP usually provide to a device?
4. Why can two devices not use the same IP address on the same network at the same time?

## Part 3: Matching Activity

Match the term to its role:

| Term | Role |
|------|------|
| DNS | A. Assigns network settings to a device |
| DHCP | B. Translates domain names into IP addresses |
| IP address | C. Identifies a device on a network |

## Part 4: Real-World Scenario

A student can open `https://google.com`, but `https://my-class-project.example` does not load.

Discuss with a partner:
1. Could DNS be part of the problem? Why?
2. Could the server still be down even if DNS works?
3. What information would you want to check next?
